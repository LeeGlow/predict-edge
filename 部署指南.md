# 部署指南 - PredictEdge

> 零成本部署 PredictEdge 到外网，任何人都能访问

## 🎯 部署架构

```
用户访问 predictedge.vercel.app
        ↓
   Vercel (前端)
        ↓
   Render (后端API)
        ↓
   Polymarket 数据源
```

**费用：0元（免费套餐足够起步）**

---

## 🚀 第一步：部署后端（Render）

### 1. 准备工作

1. 注册 [Render.com](https://render.com) 账号（免费）
2. 把代码推到 GitHub 仓库
3. 后端代码在 `backend/` 目录

### 2. 创建 Web Service

1. 登录 Render → 点击 **"New +"** → **"Web Service"**
2. 选择你的 GitHub 仓库
3. 配置：
   - **Name**: `predict-edge-backend`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app:app --host 0.0.0.0 --port $PORT`
   - **Plan**: 选 **Free**（免费）

4. **环境变量**（Advanced → Environment Variables）：
   ```
   SECRET_KEY = 你的随机密钥（随便填一串长字符串）
   USDT_WALLET = 你的USDT收款地址（TRC20）
   AUTO_VERIFY = false
   ```

5. 点击 **"Create Web Service"**

### 3. 等待部署完成

- 大概需要 2-5 分钟
- 部署成功后会得到一个 URL，类似：`https://predict-edge-backend.onrender.com`
- 测试一下：访问 `https://你的域名/health`，应该返回 JSON

---

## 🌐 第二步：部署前端（Vercel）

### 1. 准备工作

1. 注册 [Vercel.com](https://vercel.com) 账号（免费）
2. 代码已经在 GitHub 仓库

### 2. 导入项目

1. 登录 Vercel → **"Add New..."** → **"Project"**
2. 选择你的 GitHub 仓库
3. 配置：
   - **Project Name**: `predict-edge`
   - **Framework Preset**: `Other`
   - **Root Directory**: `frontend`
   - **Build Command**: 留空（纯静态文件）
   - **Output Directory**: 留空

4. 点击 **"Deploy"**

### 3. 配置 API 代理（可选，如果想统一域名）

编辑 `vercel.json` 文件，把后端地址改成你的 Render 地址：

```json
{
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "https://你的-render-地址.onrender.com/api/$1"
    }
  ]
}
```

这样访问 `https://你的项目.vercel.app/api/xxx` 就会转发到后端。

## ✅ 第三步：测试

1. 打开 Vercel 给你的域名，比如：`https://predict-edge.vercel.app`
2. 应该能看到网站界面
3. 点"注册"，创建一个账号试试
4. 登录后应该能看到事件列表

## 🎨 第四步：绑定自定义域名（可选）

### 购买域名

- 推荐：[Namecheap](https://namecheap.com)、[Cloudflare](https://cloudflare.com)
- 价格：$10-20/年
- 选一个好记的域名，比如 `predictedge.io`、`predictedge.app`

### Vercel 绑定域名

1. Vercel → Settings → Domains → Add
2. 输入你的域名，比如 `app.predictedge.io`
3. 按照提示配置 DNS 解析
4. 等几分钟，HTTPS 会自动配置好

---

## 💰 第五步：配置收款

### USDT 收款（最简单）

1. 准备一个 TRC20 钱包地址（用 OKX 或 TP 钱包）
2. 在 Render 后台设置环境变量：
   ```
   USDT_WALLET = 你的USDT地址
   ```
3. 用户付款后，你手动检查链上交易，确认后激活账号

### 自动验证（进阶，后面再做）

1. 接入 USDT 链上监听
2. 用户转账后自动激活
3. 需要：自己写监听脚本，或者用第三方服务

---

## 📊 第六步：接入真实数据源

### 当前状态

- 默认用模拟数据（8个事件）
- 已经写好了 Polymarket API 的接入代码
- 但实际可用的事件可能需要自己筛选

### 启用真实数据

1. 在 Render 环境变量设置：
   ```
   USE_REAL_DATA = true
   ```
2. 重启服务
3. 会自动从 Polymarket 拉取真实事件

### 注意事项

- Polymarket API 有速率限制，不要请求太频繁
- 免费版每分钟更新一次就够了
- 如果数据拉取失败，会自动降级到模拟数据

## 🔧 常见问题

### Q: Render 免费版会休眠怎么办？

A: 免费版 15 分钟无请求会休眠，第一次打开会慢几秒。可以用 UptimeRobot 定时 ping 一下保持唤醒。

### Q: 数据安全吗？

A: 密码是 bcrypt 哈希存储的，数据库在 Render 内部。但免费版数据不持久，重启会丢数据。生产环境建议接 PostgreSQL。

### Q: 能接 Stripe 信用卡支付吗？

A: 可以，但需要海外公司主体。先从 USDT 开始最简单。

### Q: 用户多了免费版不够用怎么办？

A: Render 付费版 $7/月起，Vercel 付费版 $20/月起。有 100 个付费用户就完全覆盖了。
