## 🚀 部署步骤详解

### 第一步：准备 GitHub 仓库

1. 在 GitHub 创建一个新仓库（比如 predict-edge）
2. 把本地代码推上去：
   ```bash
   cd d:\哈马斯\predict-edge
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/predict-edge.git
   git push -u origin main
   ```

---

### 第二步：部署后端（Render）

#### 1. 注册 Render 账号
- 访问 https://render.com
- 用 GitHub 账号登录（免费）

#### 2. 创建 Web Service
1. 点击右上角 **"New +"** → **"Web Service"**
2. 选择你刚创建的 GitHub 仓库
3. 配置：
   - **Name**: `predict-edge-backend`（随便起，会影响子域名）
   - **Runtime**: `Python 3`
   - **Region**: `Oregon`（选离你用户近的，美国用户多选 Oregon）
   - **Branch**: `main`
   - **Root Directory**: `backend`（⚠️ 重要！后端在 backend 目录里）
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn app:app --host 0.0.0.0 --port $PORT`
   - **Plan**: **Free**（先免费，后面有收入再升级）

#### 3. 配置环境变量
点击 **"Advanced"** → **"Add Environment Variable"**，添加以下变量：

| Key | Value |
|-----|-------|
| `SECRET_KEY` | 生成一个随机密钥（比如用这个：https://randomkeygen.com/ 选 256-bit WEP Key） |
| `USDT_WALLET` | `TV2MhRQDgBfFXXLxqZ58HhkGSMasCtfZPB` |
| `AUTO_VERIFY` | `false` |
| `PYTHON_VERSION` | `3.11.9` |

#### 4. 点击 "Create Web Service"
- 等待 2-5 分钟部署
- 部署成功后，你会得到一个 URL，比如：`https://predict-edge-backend.onrender.com`

#### 5. 测试后端
访问：`https://你的域名.onrender.com/health`
应该返回：
```json
{"status": "ok"}
```

再测试事件接口：
访问：`https://你的域名.onrender.com/api/events?limit=5`
应该返回事件列表的 JSON

---

### 第三步：部署前端（Vercel）

#### 1. 注册 Vercel 账号
- 访问 https://vercel.com
- 用 GitHub 账号登录（免费）

#### 2. 配置 vercel.json
**重要：** 先修改 `vercel.json` 里的后端地址，把它改成你刚部署的 Render 地址：

```json
{
  "version": 2,
  "buildCommand": "echo 'Static build complete'",
  "outputDirectory": "frontend",
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "https://你的-render-地址.onrender.com/api/:path*"
    }
  ]
}
```

把 `destination` 里的地址改成你第二步得到的 Render 后端地址。

修改后提交到 GitHub：
```bash
git add vercel.json
git commit -m "Update backend API URL"
git push
```

#### 3. 导入项目
1. 登录 Vercel → 点击 **"Add New..."** → **"Project"**
2. 选择你的 GitHub 仓库
3. 配置：
   - **Project Name**: `predict-edge`（会影响子域名）
   - **Framework Preset**: `Other`
   - **Root Directory**: 留空（因为 vercel.json 里已经配置了 outputDirectory）
   - **Build Command**: 留空（vercel.json 里配置了）
   - **Output Directory**: 留空（vercel.json 里配置了）

#### 4. 点击 "Deploy"
- 等待 1-2 分钟
- 部署成功后，你会得到一个 URL，比如：`https://predict-edge.vercel.app`

#### 5. 测试前端
打开 Vercel 给你的域名，检查：
- [ ] 页面正常加载
- [ ] 事件列表正常显示
- [ ] 注册功能正常
- [ ] 登录功能正常
- [ ] 各个标签页切换正常

---

### 第四步：绑定自定义域名（可选但推荐）

#### 为什么要绑定自定义域名？
- 更专业，让用户更信任
- 更好记，方便推广
- 可以用自己的品牌

#### 1. 购买域名
推荐平台：
- **Namecheap**: https://namecheap.com（便宜，支持支付宝）
- **Cloudflare**: https://cloudflare.com（CDN+DNS，免费）
- **阿里云/腾讯云**: 国内用户方便，但需要备案

价格：$10-20/年，推荐选 `.app` 或 `.io` 后缀，比较有科技感。

#### 2. 在 Vercel 绑定域名
1. 进入 Vercel 项目 → **Settings** → **Domains**
2. 点击 **"Add"**
3. 输入你的域名，比如 `app.predictedge.io`
4. 按照提示配置 DNS 解析

#### 3. 配置 DNS
如果用 Cloudflare：
1. 登录 Cloudflare
2. 添加你的域名
3. 添加一条 CNAME 记录：
   - **Type**: CNAME
   - **Name**: app（或者你想用的前缀）
   - **Target**: `predict-edge.vercel.app`（你的 Vercel 域名）
   - **Proxy status**: 开启（橙色云朵，享受 CDN）

#### 4. 等待生效
- DNS 解析生效需要几分钟到几小时
- Vercel 会自动配置 HTTPS 证书
- 生效后用你的域名访问，应该能看到网站

### 第五步：配置 Telegram Bot 推送（可选）

#### 1. 创建 Telegram Bot
1. 在 Telegram 里找 **@BotFather**
2. 发 `/newbot`，按照提示创建
3. 得到 Bot Token，保存下来

#### 2. 获取 Chat ID
1. 在 Telegram 里找 **@userinfobot**
2. 发 `/start`，它会告诉你你的 ID

#### 3. 配置环境变量
在 Render 后台添加环境变量：
- `TELEGRAM_BOT_TOKEN`: 你的 Bot Token
- `TELEGRAM_CHAT_ID`: 你的 Chat ID

重启服务后，新的套利信号会自动推送到 Telegram。

## 🔥 紧急情况处理

### 情况1：网站打不开了
1. 先检查是不是你自己的网络问题
2. 访问 https://你的域名/health，看后端是不是正常
3. 登录 Render/Vercel 后台看日志
4. 如果后端挂了，试试重启服务
5. 如果还是不行，回滚到上一个版本

### 情况2：数据库出问题了
1. 先备份现有数据库
2. 如果是 SQLite，直接下载 db 文件备份
3. 如果数据丢了，用备份恢复
4. 用户多了建议换成 PostgreSQL

### 情况3：数据源挂了
1. Polymarket/Manifold API 可能偶尔挂
2. 代码里已经有降级处理，会用模拟数据兜底
3. 等 API 恢复了就正常了
4. 如果经常挂，考虑加缓存

### 情况4：有人恶意注册/攻击
1. 先看日志，找到异常 IP
2. Render 后台可以配置 IP 黑名单
3. 考虑加验证码（后面优化）
4. 严重的话暂时关闭注册
